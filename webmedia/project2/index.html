Now, they look a lot better.

How did that happen? </p>

2002: Audi

The beginnings of responsive design principles can be traced back to a redesign of the Audi website, which launched in 2002. The website used JavaScript on each page to determine the size of the user's viewport, and could dynamically adjust to one of three layout sizes.

2010: Ethan Marcotte

Web designer Ethan Marcotte was the first to coin the term "Responsive Web Design" in 2010. In an article he wrote for A List Apart, Marcotte named three pre-existing technical components that he viewed as essential to this new design ideology: fluid grids, responsive images, and media queries. "Now, more than ever," he wrote, "we're designing work meant to be viewed along a gradient of different experiences."

2011: Bootstrap

Designer Mark Otto and developer Jacob Thornton created Bootstrap (originally Twitter Blueprint) at Twitter in mid-2010. Originally an internal project, it was made open source in 2011 and continues to be maintained by its original development team. Bootstrap 2.0, introduced in early 2012, included a responsive 12-column grid system. The project since has been a massive success due to its ease of use, and is the fourth-most forked project of all time on GitHub.

2015: Mobilegeddon

In April of 2015, Google put pressure on web designers to adopt new standards of responsive design when it updated its search algorithm to prioritize responsive sites in its rankings if the search originated from a mobile device. This cemented responsive design as the new standard for web development: web developers now had to utilize it in order to push their sites up in the rankings.

2016: Revolution

For the first time in history, page views from mobile devices overtook page views from desktop computers in October 2016, largely due to the ease of use that responsive design provides for any user on any device - regardless of viewport size.
